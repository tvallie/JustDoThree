# JustDoThree v2.1 Task Date Test Plan

## Goal
Verify that the optional Task Date feature behaves correctly when enabled, stays invisible when disabled, preserves stored data across feature-flag changes, and does not change existing Today, plan, reminder, or backlog behavior outside the intended UI surfaces.

## Test Areas

### Settings Toggle
- Fresh install or clean defaults: `Enable Task Dates` is off.
- Toggle off: no Task Date UI appears in add, edit, or backlog.
- Toggle on: add/edit controls and backlog display become available.
- Toggle off after dates exist: UI hides but stored values remain.
- Toggle back on: previously saved dates reappear.

### Task Creation And Editing
- Create task with no Task Date: task saves and shows no date line in backlog.
- Create task with today’s date: task saves and date appears in backlog only.
- Create task with a future date: task saves and displays correctly.
- Edit task to add a date: backlog updates after save.
- Edit task to change a date: backlog updates after save.
- Edit task to remove a date: date line disappears.
- Past dates cannot be selected in the picker.
- Existing tasks with no date can still be edited without changing recurrence behavior.

### Backlog UI
- Created date is no longer visible anywhere in backlog.
- Task Date uses subtle secondary styling and no urgency language.
- Tasks with no date show no placeholder text.
- Existing rollover metadata still displays correctly.
- Existing recurring metadata still displays correctly.
- Existing future-plan notes still display correctly.
- Search and manual reorder continue to work exactly as before.

### Feature Flag Edge Cases
- If Settings is turned off while `AddTaskSheet` is open, Task Date controls hide immediately.
- Turning the feature off while the sheet is open does not delete already-saved dates.
- Unsaved in-progress picker changes are not persisted unless the user saves.
- If older or seeded data contains a past stored `taskDate`, backlog displays it neutrally when the feature is enabled.

### Regression Checks
- Today view remains visually and behaviorally unchanged.
- Plan view remains unchanged.
- History and stats views remain unchanged.
- Notifications and reminders remain unchanged.
- Rollover behavior remains unchanged.
- Recurring task behavior remains unchanged.
- Paste import and file import flows remain unchanged.
- No new sorting or filtering occurs based on Task Date.
- No new analytics, scoring, or streak behavior is introduced.

## Suggested Verification Order
1. Turn the feature on and verify create, edit, and backlog display behavior.
2. Turn the feature off and verify the UI disappears cleanly.
3. Turn the feature back on and confirm previously saved dates return.
4. Run a quick regression pass through Today, Plan, rollover, and recurring flows.

## Current Verification Status
- Code compiles successfully with the feature changes.
- Simulator/device UI verification is still pending.
