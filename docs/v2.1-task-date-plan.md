# JustDoThree v2.1 Task Date Implementation Plan

## Summary
Add an optional, neutral `Task Date` feature that is fully hidden behind a new Settings toggle and visible only in backlog plus create/edit task controls. The feature stores a date-only value on each task, allows today-or-future selection, preserves data when toggled off, and has no behavioral impact on Today, planning, reminders, analytics, sorting, or scoring.

## Implemented Changes
- Data model
  - Added nullable `taskDate: Date?` to `JDTask`.
  - New tasks default `taskDate` to `nil`.
  - Add/edit saves normalize selected values to `startOfDay` for date-only semantics.
  - Existing tasks are left unchanged with `taskDate = nil`.
  - `createdDate` remains stored, but backlog UI no longer displays it.

- Settings
  - Added `@AppStorage("jdt_enableTaskDates")` toggle to `SettingsView`.
  - Label is `Enable Task Dates`.
  - Default remains `false`.
  - Toggle sits in the existing `Features` section below recurring-task settings.

- Create/edit task flow
  - Extended `AddTaskSheet` with task-date UI only when the setting is on.
  - Added `@AppStorage("jdt_enableTaskDates")` plus local state for `taskDateEnabled` and `selectedTaskDate`.
  - In create mode:
    - default state is unset
    - picker is shown only after tapping `Add Task Date`
    - picker uses native `.date`
    - selection is limited to `Date().startOfDay...`
    - saving without a selected date keeps `taskDate = nil`
  - In edit mode:
    - existing `taskDate` preloads if present
    - add, change, and remove are supported
  - Clear affordance:
    - if no date exists, show `Add Task Date`
    - once enabled, show the picker plus `Remove Task Date`
  - Save logic:
    - persists normalized `startOfDay` when selected
    - persists `nil` when removed or never set

- Backlog display
  - Updated `BacklogRow` to conditionally render task metadata based on the task-date feature flag.
  - When Task Dates are enabled and `task.taskDate != nil`:
    - shows the formatted date as the first secondary line beneath the title
    - uses subtle caption styling with no urgency language
  - When Task Dates are enabled and `task.taskDate == nil`:
    - shows no date text
  - When Task Dates are disabled:
    - shows no task-date UI
  - Removed `createdDate.monthDayString` from backlog row UI.
  - Kept rollover, recurring, and future-plan notes as secondary metadata.
  - Added a helper that shows abbreviated month/day and includes year only when the date falls outside the current year.

## Scope Protection
- Task Date is still intentionally absent from:
  - `TodayView`
  - `PlanView`
  - history/stats views
  - notifications/reminders
  - rollover logic
  - backlog picker rows
- No sort/filter logic has been tied to `taskDate`.
- Today selection, completion, streaks, analytics, and recurring behavior were left untouched.

## Public API / Type Changes
- `JDTask`
  - Added `var taskDate: Date?`
- `AddTaskSheet`
  - No external API change required.
  - Internal save/load logic now handles optional `taskDate`.
- `BacklogRow`
  - Added feature-flagged task-date rendering and removed created-date UI.

## Assumptions
- Task Date is stored directly as `Date?` on `JDTask`, not encoded into a secondary JSON blob.
- Locale-aware abbreviated month/day is the preferred display format, with year shown only when needed for clarity.
- The clear/remove affordance in create/edit remains a plain text button within the same section.
- Backlog shows Task Date only when the feature is enabled; when disabled, the row does not reserve empty space for it.
